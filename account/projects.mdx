Projects are the simplest way to keep your work organized in Uxia. Think of them as basic folders: each project groups together the tests you launch, so related work stays in one place and is easy to find later. Every test lives inside a project, and you can create as many projects as you need to match how your team works — by product area, client, sprint, or anything else.

## Creating a project

You can create a project at two moments:

- **Ahead of time**, from the Projects area, so it's ready before you start testing.
- **While creating a test**, by adding a new project inline during setup and assigning the test to it right away.

Either way, the project is available immediately for any future tests.

## Organizing your tests

Projects are flexible — nothing is locked in place once a test is running.

- **Move tests between projects at any point.** If a test ends up in the wrong place, or your organization changes, you can reassign it to a different project whenever you like.
- **Keep related tests together** so results, transcripts, and insights for a given initiative are grouped rather than scattered across your workspace.

## Archiving a project

When a project is finished or no longer active, you can **archive** it to keep your workspace tidy without losing anything. Archived projects and their tests stay accessible for reference — they're simply moved out of your main view.
